# Maia Hangman Game

A real-time, multiplayer Hangman web application where two players interact synchronously — one sets the word and the other tries to guess it.

## About This Project

This entire codebase was **generated by an AI agent** ([Abacus AI Agent](https://abacus.ai)) from a single architecture design prompt. The prompt that produced this application is included in the repository as [`ARCHITECTURE_PROMPT.md`](./ARCHITECTURE_PROMPT.md). It defines the complete system specification — domain model, game logic, scoring rules, UI components, and architectural principles — which the agent then used to build the full-stack application.

## How It Works

The game supports two roles:

- **Word-Setter (Host)** — Creates a game, defines a word or phrase, provides three hints, and optionally customizes scoring penalties.
- **Word-Guesser (Player)** — Joins via a unique Game ID and attempts to solve the puzzle by guessing letters or the full word.

### Game Flow

1. The Host creates a new game and receives a unique 20-character alphanumeric Game ID.
2. The Host configures the puzzle: word/phrase, three ordered hints (hard → easy), and optional scoring overrides.
3. The Guesser joins by entering the Game ID and a player name.
4. The Host starts the game. The Guesser guesses letters using an on-screen keyboard or submits a full-word guess.
5. The game ends when the word is fully revealed, the Guesser submits the correct full word, or the score drops to zero.

### Scoring

| Action | Penalty |
|---|---|
| Incorrect consonant guess | `100 / word_length` |
| Incorrect vowel guess | `2 × (100 / word_length)` |
| Hint request | 15 points |

The starting score is **100 points**. The Host can override all penalty values during game setup.

## Key Design Decisions

- **Pure Business Logic** — All game mechanics (letter masking, penalty calculation, win/loss checks) live in `lib/game-logic.ts` as pure, side-effect-free functions, separate from I/O and state management.
- **REST Polling over WebSockets** — Real-time synchronization uses a 1.5-second polling interval instead of WebSockets for full compatibility with serverless deployment environments.
- **Session Persistence** — Players can reconnect to an active game after a browser refresh using `localStorage`-backed session recovery.
- **Punctuation Handling** — Punctuation and spaces in phrases are automatically revealed in the word display; only alphabetic characters require guessing.

## Getting Started

1. **Clone the repository**

   ```bash
   git clone https://github.com/aguiguy/maia_hangman_game.git
   cd maia_hangman_game
   ```

2. **Install dependencies**

   ```bash
   yarn install
   ```

3. **Configure environment variables**

   Create a `.env` file in the project root:

   ```env
   DATABASE_URL="postgresql://user:password@localhost:5432/hangman?connect_timeout=15"
   ```

4. **Set up the database**

   ```bash
   yarn prisma generate
   yarn prisma db push
   ```

5. **Run the development server**

   ```bash
   yarn dev
   ```

   Open [http://localhost:3000](http://localhost:3000) in your browser.
